# ARCHived - Implementation Plan

## Context
Build a "killedbygoogle" style static website that showcases archived/unmaintained GitHub OSS projects. The goal is to give visibility to dead projects so others can potentially fork/revive them.

## Architecture
- **Static site** (HTML/CSS/JS) generated from JSON data
- **GitHub Actions** for weekly automated scanning and site rebuild
- **GitHub Pages** for hosting (free, simple)
- **Python scraper** to collect dead projects from GitHub API

## Data Collection Logic

### What qualifies as "dead" (Both conditions required):
1. **Archived by maintainers**: GitHub's `archived` flag is true
2. **No recent activity**: No commits in the last 12 months (configurable)

### Data points to collect per repository:
- Name (owner/repo)
- Description
- URL
- Stars count (for popularity metric)
- Primary language
- Last commit date
- Archived date (if available from API)
- Topics/tags
- License (if available)

### Discovery approach:
- Use GitHub Search API: `archived:true` + `pushed:<date>` filter
- Search query: `archived:true pushed:<=2025-03-29` (dynamic date calculation)
- Process results in batches (handle pagination)
- Optional: also include a `dead-projects.json` for manual additions/submissions

### Rate limiting considerations:
- GitHub API: 30 req/min for unauthenticated, 5000/hr for authenticated
- Use `GITHUB_TOKEN` secret in Actions
- Implement retry logic and sleep between pages
- Store results incrementally if needed

## UI/UX Design

### Layout Options:
1. **List view** (simple): One project per row with key info
2. **Card grid** (more visual): Cards showing name, description, stars, language
3. **Hybrid**: Grid of cards with sort/filter controls above

**Chosen approach**: Card grid similar to killedbygoogle but cleaner, with:
- Sort by: stars (default), last commit date, archived date
- Filter by: language, star range, date range
- Responsive grid layout

### Card content:
- Project name (linked)
- Short description (truncated if long)
- Stats: ⭐ stars, 🏷️ language, 📅 last commit X months ago
- "View on GitHub" button
- Maybe: "Suggest revive" link (optional future feature)

### Styling:
- Use CSS Grid for responsive layout
- Minimal color palette: white/gray + accent color (maybe GitHub-like orange or purple)
- Mobile-first responsive design
- Load `dead-projects.json` via fetch() in JavaScript

## GitHub Actions Workflow

### Schedule:
- Weekly: `cron: '0 0 * * 0'` (Sunday midnight UTC)

### Steps:
1. **Checkout** repository
2. **Setup Python** (3.11 or 3.12)
3. **Install dependencies** (requests, python-dateutil)
4. **Run scraper**: `python scraper/collect.py`
   - Outputs `dead-projects.json`
   - Also outputs `dead-projects.json` to `docs/` folder for Pages
5. **Generate site**: Simple script that copies HTML/CSS/JS to docs/ + embeds/links JSON
6. **Commit changes** if data changed
7. **Deploy** to GitHub Pages from `docs/` folder

### Secrets:
- `GITHUB_TOKEN`: Use default `GITHUB_TOKEN` (automatically provided)

### Error handling:
- Fail workflow if scraper encounters API errors after retries
- Notify via Actions logs; optionally send email on failure

## Implementation Steps (Order)

1. **Create base HTML/CSS/JS** - the static site skeleton
2. **Write scraper script** - fetch archived repos from GitHub API
3. **Create GitHub Actions workflow** - automate weekly scans
4. **Configure GitHub Pages** - point to `docs/` branch/folder
5. **Test** - run scraper manually, verify site builds
6. **Manual test data** - add a few sample repos to dead-projects.json for initial design
